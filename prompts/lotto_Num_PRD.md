# 로또 추천번호 시스템 PRD
**Lotto Number Recommendation System**
**Product Requirements Document v1.0**
작성일: 2026-05-28

---

## 1. 문서 개요

### 1.1 목적
본 문서는 바닐라 JavaScript(HTML/CSS/JS)만을 사용하여 구현하는 로또 추천번호 시스템의 기능 요구사항, UX 흐름, 기술 스펙을 정의한다.

### 1.2 제품 비전
사용자가 직접 제외 번호와 포함 번호를 설정하여 수동·반자동 방식으로 로또 번호를 추천받을 수 있는 인터랙티브 웹 애플리케이션

### 1.3 범위

| 구분 | 내용 |
|------|------|
| In Scope | 번호 선택 UI, 제외/추천 번호 설정, 랜덤 추천 생성, 결과 출력 |
| Out of Scope | 당첨 이력 저장, 회차 통계 API 연동, 로그인/계정 관리 |
| 기술 스택 | Vanilla HTML5 / CSS3 / JavaScript (ES6+), 외부 라이브러리 미사용 |

## 2. 사용자 요구사항

### 2.1 타겟 사용자
- **일반 로또 구매자**: 매주 번호를 고르는 번거로움을 줄이고 싶은 사용자
- **반자동 선호자**: 특정 번호는 직접 고르되 나머지는 추천 받고 싶은 사용자
- **징크스 관리자**: 특정 번호를 절대 넣고 싶지 않은 사용자

### 2.2 핵심 사용자 스토리

| ID | 사용자 스토리 | 우선순위 |
|----|--------------|---------|
| US-01 | 나는 특정 번호를 제외 목록에 추가하여 추천 결과에서 해당 번호가 나오지 않도록 하고 싶다. | High |
| US-02 | 나는 반드시 포함되길 원하는 번호를 지정하고 나머지는 자동 추천 받고 싶다. | High |
| US-03 | 추천 버튼 클릭 한 번으로 유효한 6개 번호 세트를 즉시 확인하고 싶다. | High |
| US-04 | 선택한 번호(제외/포함)를 한 눈에 확인하고 개별 취소할 수 있어야 한다. | Medium |
| US-05 | 결과 번호가 색상 공으로 시각적으로 표시되어 실제 로또와 유사하게 보이면 좋겠다. | Medium |
| US-06 | 전체 초기화 버튼으로 모든 선택을 한 번에 리셋할 수 있어야 한다. | Low |

## 3. 기능 요구사항

### 3.1 번호판 (Number Grid)
1~45 번호가 그리드 형태로 표시된다.

- 각 번호는 클릭 가능한 버튼으로 렌더링
- 클릭 시 현재 활성 모드(제외/포함)에 따라 상태 변경
- 번호 색상은 10단위 구간별로 구분
- 상태별 시각적 구분

| 상태 | 표시 방식 |
|------|----------|
| 기본 | 흰 배경, 구간 색상 텍스트 |
| 제외 | 빨간 배경 + X 표시 |
| 포함 | 파란 배경 + ✓ 표시 |

### 3.2 제외 번호 기능 (Exclude Mode)
- "제외 번호 선택" 모드 활성화 후 번호 클릭 시 제외 등록
- 제외 번호는 최대 39개까지 설정 가능 (최소 6개는 남아야 추천 가능)
- 제외된 번호는 추천 대상 풀(pool)에서 완전히 제거
- 선택된 제외 번호 목록을 태그(badge) 형태로 표시, 개별 클릭으로 해제 가능

### 3.3 추천 포함 번호 기능 (Include Mode)
- "포함 번호 선택" 모드 활성화 후 번호 클릭 시 포함 번호 등록
- 포함 번호는 최대 6개까지 설정 가능
- 포함 번호로 지정된 번호는 제외 번호와 중복 설정 불가 (UI에서 방지)
- 포함 번호가 6개 미만일 경우 나머지는 랜덤으로 채움
- 포함 번호가 정확히 6개면 추천 버튼 클릭 시 해당 번호를 그대로 결과로 표시

### 3.4 추천 번호 생성 (Recommendation Engine)
- "추천 번호 생성" 버튼 클릭 시 동작
- 생성 로직:
  - 가용 풀 = 전체(1~45) − 제외 번호
  - 결과 = 포함 번호 + 랜덤(가용 풀에서 `6 - 포함수` 만큼 비복원 추출)
  - 결과 6개를 오름차순 정렬 후 출력
- 가용 번호가 6개 미만이면 경고 메시지 표시 후 생성 중단
- 동일 버튼 재클릭 시 새로운 추천 결과 재생성

### 3.5 결과 출력
- 추천된 6개 번호를 로또 공 스타일의 원형 UI로 표시
- 각 공은 번호 구간에 따른 배경색 적용
- 결과 영역은 생성 전 숨김 처리, 생성 후 애니메이션과 함께 노출
- "다시 뽑기" 버튼으로 즉시 재생성 가능

### 3.6 초기화
- "전체 초기화" 버튼으로 제외 번호, 포함 번호, 결과 모두 리셋
- 초기화 전 확인 다이얼로그(`confirm`) 표시

## 4. UX 흐름 (User Flow)

### 4.1 전체 흐름

```
[진입] 1~45 번호 그리드 렌더링
   ↓
[모드 선택] "제외" 또는 "포함" 버튼 클릭
   ↓
[번호 클릭] 선택한 모드에 따라 번호 상태 변경 + 태그 목록 업데이트
   ↓
[추천 생성] "추천 번호 생성" 버튼 클릭 → 유효성 검사
   ↓
[결과 출력] 6개 공이 순차 애니메이션으로 표시
   ↓
[재생성 or 초기화] "다시 뽑기" / "전체 초기화"
```

| 단계 | 사용자 액션 | 시스템 반응 |
|------|------------|------------|
| 1 | 페이지 진입 | 1~45 번호 그리드 렌더링, 모든 상태 초기화 |
| 2 | 모드 선택 | "제외" 또는 "포함" 모드 버튼 클릭 → 활성 모드 하이라이트 |
| 3 | 번호 클릭 | 선택한 모드에 따라 해당 번호 상태 변경 + 목록 업데이트 |
| 4 | 추천 생성 | "추천 번호 생성" 버튼 클릭 → 유효성 검사 → 결과 출력 |
| 5 | 결과 확인 | 6개 공이 순차 애니메이션으로 표시됨 |
| 6 | 재생성 or 초기화 | "다시 뽑기" 재생성 / "전체 초기화" 리셋 |

### 4.2 예외 처리 흐름

| 예외 상황 | 처리 방식 |
|----------|----------|
| 가용 번호 < 6개 | 빨간 경고 메시지: "추천 가능한 번호가 부족합니다" |
| 포함 번호 > 6개 시도 | 7번째 클릭 시 무시 + 노란 안내: "최대 6개까지 선택 가능합니다" |
| 포함 번호가 제외 번호와 중복 | 클릭 자체를 비활성화(disabled) 처리 |
| 아무 설정 없이 생성 | 정상 동작 (완전 랜덤 6개 추천) |

## 6. 기술 요구사항

### 6.1 기술 스택
- **HTML5**: 시맨틱 마크업, 단일 `.html` 파일로 완결
- **CSS3**: Flexbox / Grid 레이아웃, CSS Variables, Transition/Animation
- **JavaScript ES6+**: 외부 라이브러리 없이 Vanilla JS만 사용
- 별도 빌드 도구 없이 브라우저에서 직접 실행 가능

### 6.2 핵심 JS 모듈 구조

| 모듈/함수 | 역할 |
|----------|------|
| `state` 객체 | `excludeSet(Set)`, `includeSet(Set)`, `currentMode(string)` 관리 |
| `renderGrid()` | 1~45 번호 버튼 DOM 생성 및 초기 렌더링 |
| `handleNumberClick(num)` | 현재 모드에 따라 exclude/include Set 업데이트 |
| `updateBadges()` | 제외/포함 태그 목록 DOM 동기화 |
| `generateNumbers()` | 추천 로직 실행 → 결과 배열 반환 |
| `renderResult(numbers)` | 결과 공 UI 렌더링 + 애니메이션 트리거 |
| `resetAll()` | State 초기화 + 전체 UI 리렌더링 |

### 6.3 데이터 구조

```javascript
// 전역 상태
const state = {
  excludeSet: new Set(),   // 제외 번호 집합
  includeSet: new Set(),   // 포함 번호 집합
  mode: 'exclude'          // 'exclude' | 'include'
};

// 추천 생성 로직
function generateNumbers() {
  const pool = Array.from({ length: 45 }, (_, i) => i + 1)
    .filter(n => !state.excludeSet.has(n) && !state.includeSet.has(n));

  const needed = 6 - state.includeSet.size;

  if (pool.length < needed) {
    showError("추천 가능한 번호가 부족합니다.");
    return null;
  }

  // Fisher-Yates shuffle 후 needed개 추출
  const picked = shuffle(pool).slice(0, needed);
  return [...state.includeSet, ...picked].sort((a, b) => a - b);
}
```
